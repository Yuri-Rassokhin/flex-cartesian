Explore real systems as functions of parameters.

Build a holistic digital twin in four steps: describe parameters, model system behaviour, capture real data, and derive insights. From bottlenecks to best practices — and beyond. Any question, answered in one line of code.

Taming fragmented knowledge.

Real systems are defined by multiple independent parameters. Each configuration is just one observation in a much larger space. Yet exploration is typically performed through isolated runs: ad-hoc benchmarks, biased experiments, manual comparisons, and fragmented logs. These observations are never integrated into a coherent model. What we get is not understanding — but a collection of disconnected observations. The system is never seen as a whole — rather built upon assumptions.

From scattered probes to exploration.

What if we stopped running experiments… and started exploring the system itself? FlexCartesian treats a system not as a set of scattered measurements, but as a holistic function that behaves over space with dimenions defined by system parameters. This shift in perspective from blind probes to parameteric analysis allows us to move from fragmented data points to a structured understanding of how the system behaves across different configurations. It enables us to identify patterns, trade-offs, and bottlenecks that would be invisible in isolated runs.

Concise for human, precise for insight.

The system is evaluated across the entire parameter space. Each result becomes part of a structured behavioural model — not just an individual measurement. Everything in high-level declarative style.

require 'flex-cartesian'

space = FlexCartesian.new({count:[2, 4], size:[32, 64], target:["8.8.8.8", "1.1.1.1", "208.67.222.222"]})

result = {}

space.func(:add, :command) { |v| "ping -c #{v.count} -s #{v.size} #{v.target}" }
space.func(:add, :raw, hide: true) { |v| result[v.command] ||= `#{v.command} 2>&1` }
space.func(:add, :time) { |v| v.raw[/min\/avg\/max\/(?:mdev|stddev) = [^\/]+\/([^\/]+)/, 1]&.to_f&.round(2) }

space.func(:run, progress: true, title: "Obtaining data")

space.output(colorize: true)

Complete behavioural model.

Instead of isolated runs, FlexCartesian builds a continuous behavioural model of the system across its entire parameter space. It reveals how performance evolves across dimensions, where optimal configurations emerge, and where limits, bottlenecks, and trade-offs define the system's behaviour.

This makes optimisation systematic rather than experimental, exposes non-obvious interactions between parameters, and makes system behaviour reproducible and explainable.

The system is no longer sampled. It is understood.

Genuinely holistic system exploration.

FlexCartesian provides a new way to model systems: not as isolated experiments, but as holistic functions over comprehensive spaces of possible configurations.

What makes it different

  • High-level human-readable DSL
  • Systems as first-class parametric spaces
  • Separation of system definition from execution
  • Functions over parameter space as the core abstraction
  • Structured reasoning about system behaviour
  • Real-world measurements integrated into the model

Born in field work

FlexCartesian emerged from real-world AI infrastructure work, where behaviour depends on multiple interacting parameters: hardware, storage, network, and model configuration.

Traditional tools failed to provide a unified abstraction for exploring such systems as a whole.

Applied in critical systems

  • Performance modelling across distributed systems
  • AI architecture benchmarking: GPU, storage, and network trade-offs
  • Architecture decision support for enterprise AI deployments

What this enables

  • Discovery of optimal configurations without exhaustive search
  • Identification of system bottlenecks across parameters
  • Alloy of theoretical modelling with real measurements
